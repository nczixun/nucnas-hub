# 内容生成工作流程

## 1. 监控发现

### 每日任务
- [ ] 搜索 B站 迷你主机最新测评
- [ ] 搜索 B站 NAS 测评
- [ ] 收集新品发布信息

### 关键词列表
```
site:bilibili.com 迷你主机 测评 2026
site:bilibili.com NAS 测评 2026
迷你主机 推荐 2026年
NAS 推荐 2026年
```

---

## 2. 去重检查

### 检查现有文章
```bash
# 列出所有硬件文章中的产品型号
grep -r "title:" content/hardware/*.md
```

### 对比流程
1. 提取测评视频中的产品型号
2. 在现有文章中搜索相同型号
3. 如找到重复 → 标记为"更新"
4. 如未找到 → 标记为"新建"

---

## 3. 内容生成

### 使用深度测评模板

从测评视频中提取以下信息：

| 模板字段 | 数据来源 |
|----------|----------|
| 基本信息 | 视频标题、简介 |
| 跑分数据 | 视频中展示的测试结果 |
| 温度/功耗 | 视频中实测数据 |
| 优缺点 | UP主总结 |
| 购买建议 | UP主推荐 |

### 文章结构
1. 前言（产品简介）
2. 基本参数表
3. 实测数据（核心差异化）
4. 场景测试
5. 优缺点总结
6. 购买建议
7. 竞品对比

---

## 4. 发布流程

### 检查项
- [ ] 内容无侵权（注明来源）
- [ ] 数据准确
- [ ] SEO 优化
- [ ] 格式规范

### 发布
- [ ] 推送到 GitHub
- [ ] 触发 Cloudflare 部署

## 示例：处理一个测评视频

### 视频信息
- 标题：零刻GTR7评测：迷你主机性能有上限，别再吹了！
- UP主：不满格评测
- 播放：27.3万

### 处理步骤
1. 提取型号：零刻 GTR7
2. 检查重复：搜索现有文章
3. 生成文章：按模板填充
4. 发布
