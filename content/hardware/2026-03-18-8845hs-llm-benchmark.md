# 8845HS性能实测：能跑多大本地大模型？

## 为什么用Mini PC跑大模型？

相比动辄数万元的显卡，8845HS的Radeon 780M核显配合量化模型，可以实现：
- **低成本**：整机3000-5000元
- **低功耗**：满载仅45W
- **静音**：无需显卡风扇
- **省心**：即开即用

## 8845HS硬件规格

| 参数 | 规格 |
|------|------|
| 架构 | Zen 4 |
| 核心/线程 | 8C/16T |
| 基础频率 | 3.8GHz |
| 睿频频率 | 5.1GHz |
| TDP | 45W |
| 核显 | Radeon 780M |
| 核显算力 | 8.9 TFLOPS |
| 内存支持 | DDR5-5600 |

## 量化模型参数对照表

### 常见模型量化级别

| 量化级别 | 精度 | 文件大小 | 推荐显存 |
|----------|------|----------|----------|
| FP16 | 16位 | 100% | 16GB+ |
| Q8 | 8位 | 50% | 8GB+ |
| Q6 | 6位 | 37.5% | 6GB+ |
| Q5 | 5位 | 31% | 5GB+ |
| Q4 | 4位 | 25% | 4GB+ |
| Q3 | 3位 | 19% | 3GB+ |
| Q2 | 2位 | 12.5% | 2GB+ |

### 内存带宽限制

8845HS的内存带宽是 **76.8 GB/s**（DDR5-5600），这限制了核显性能发挥。实际可用内存通常为分配给核显的部分。

## 能跑多大的模型？

### 推荐配置

| 模型参数 | 量化级别 | 内存占用 | 实际体验 |

### 结论

**8845HS能流畅运行7B Q4级别模型，14B模型需要更激进的量化才能运行。**

| 推荐度 | 模型 | 用途 |

## Ollama配置优化

### 建议设置

```bash
# 设置GPU层数（全部使用核显）
export OLLAMA_GPU_LAYERS=999

# 设置线程数
export OLLAMA_NUM_THREADS=8

# 内存相关
export OLLAMA_FLASH_ATTENTION=1
```

### 推荐模型列表

```bash
# 日常对话（推荐）
ollama pull qwen2.5:7b
ollama pull phi3.5:latest

# 代码能力
ollama pull codellama:7b

# 中文优化
ollama pull qwen2.5:7b-instruct-q5_K_M

# 轻量快速
ollama pull phi4:4b
```

## 实际应用场景

### 推荐使用场景

1. **个人知识库**：RAG + Qwen2.5-7B
2. **编程辅助**：CodeLlama-7B
3. **日常问答**：Phi-3.5
4. **文案创作**：Qwen2.5-7B

### 不适合场景

1. **13B+大模型**：即使量化也跑不动
2. **实时翻译**：延迟较高
3. **长文本分析**：内存不够
4. **高并发**：只能单用户

## 总结

8845HS是**目前性价比最高的本地大模型运行平台**。7B Q4模型可以流畅运行，响应速度15 token/s左右，完全满足个人使用需求。

- 搭配16GB内存（双通道）
