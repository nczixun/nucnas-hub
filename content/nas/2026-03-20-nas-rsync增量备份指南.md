# NAS增量备份完全指南：Rsync + Rclone实现多端同步

数据无价。作为NAS用户，你一定经历过"硬盘挂了怎么办"的灵魂拷问。本文将手把手教你搭建完整的增量备份体系。

## 备份策略：3-2-1原则

经典的3-2-1备份策略：

- **3**：至少3份数据副本
- **2**：存储在2种不同介质上
- **1**：1份异地备份

### 推荐备份架构

```
原始数据 (NAS Primary)
    ├── 本地备份 (NAS Secondary/USB硬盘)
    ├── 异机备份 (另一台NAS/服务器)
    └── 云端备份 (阿里云/Google Drive/OneDrive)
```

## Rsync本地增量备份

### 安装Rsync

```bash
# Ubuntu/Debian
sudo apt install rsync

# CentOS/RHEL
sudo yum install rsync

# 群晖NAS
# 套件中心直接安装
```

### 基础增量同步

```bash
# 最简单的增量同步
rsync -avz --progress /source/ /destination/

# 参数说明：
# -a: 归档模式（保留权限、时间等）
# -v: 显示详细过程
# -z: 压缩传输
# --progress: 显示传输进度

# 删除目标中不存在的文件
rsync -avz --delete /source/ /destination/
```

### 增量备份脚本

```bash
#!/bin/bash
# backup.sh - 智能增量备份脚本

SOURCE="/mnt/storage/important"
BACKUP="/mnt/backup/important"
LOGFILE="/var/log/backup.log"

# 创建带时间戳的备份目录
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="${BACKUP}_${DATE}"

echo "[$(date)] 开始备份: $SOURCE -> $BACKUP_DIR" | tee -a $LOGFILE

# 使用硬链接保留历史版本
# 相同文件不重复占用空间
rsync -avh --progress \
    --delete \
    --link-dest="${BACKUP}_latest" \
    "$SOURCE/" \
    "$BACKUP_DIR/" \
    2>&1 | tee -a $LOGFILE

# 更新latest软链接
rm -f "${BACKUP}_latest"
ln -s "$BACKUP_DIR" "${BACKUP}_latest"

echo "[$(date)] 备份完成" | tee -a $LOGFILE

# 清理超过30天的旧备份
find $BACKUP -maxdepth 1 -type d -name "*_*" -mtime +30 -exec rm -rf {} \;
```

### Rsync服务端配置

```bash
# 服务端配置 /etc/rsyncd.conf

[backup]
    path = /mnt/backup
    comment = Backup Module
    auth users = backupuser
    secrets file = /etc/rsyncd.secrets
    read only = false
    list = false

# 密码文件 /etc/rsyncd.secrets
# backupuser:password123
# 权限: chmod 600 /etc/rsyncd.secrets
```

### Rsync客户端配置

```bash

    --password-file=/etc/rsync_client.secrets \
    /local/data/ \
    backupuser@192.168.1.200::backup

# 密码文件 /etc/rsync_client.secrets
# password123
# 权限: chmod 600 /etc/rsync_client.secrets
```

## Rclone云端同步

### 安装Rclone

```bash
# Linux
curl https://rclone.org/install.sh | sudo bash

# 群晖NAS
# 下载对应架构的二进制文件
# https://rclone.org/downloads/
```

### 配置云端存储

```bash
# 交互式配置
rclone config

# 选择 n) New remote
# 输入名称: aliyun

# 根据指引完成配置
# 支持: 阿里云、腾讯云、Google Drive、OneDrive、S3等

# 列出已配置的云端
rclone listremotes

# 列出云端文件
rclone ls aliyun:/backup/
```

### 常用云端配置示例

```yaml
# 阿里云OSS配置
# type = oss
# provider = Aliyun
# access_key_id = your_access_key
# access_key_secret = your_secret
# region = cn-shanghai
# endpoint = 
# bucket = your-bucket-name
```

### 基础同步命令

```bash
# 同步本地到云端（单向）
rclone sync /local/path aliyun:bucket-name -P

# 同步云端到本地
rclone sync aliyun:bucket-name /local/path -P

# 双向同步
rclone bisync /local/path aliyun:bucket-name

# 常用参数
# -P: 显示实时进度
# -v: 详细输出
# --dry-run: 模拟运行
# --exclude: 排除文件
# --bwlimit: 带宽限制
# --transfers: 并发数
```

### 定时同步脚本

```bash
#!/bin/bash
# cloud_backup.sh - 云端备份脚本

SOURCE="/mnt/storage"
REMOTE="aliyun:nas-backup"
LOGFILE="/var/log/rclone_backup.log"
EMAIL="your@email.com"

# 排除不需要备份的文件
EXCLUDE_FILE="/root/.rclone_exclude.txt"

cat > $EXCLUDE_FILE <<'EOF'
- .DS_Store
- Thumbs.db
- *.tmp
- .cache/**
- Downloads/**
- temp/**
EOF

# 执行同步
echo "[$(date)] 开始云端同步" | tee -a $LOGFILE

rclone sync "$SOURCE" "$REMOTE" \
    --exclude-from=$EXCLUDE_FILE \
    --progress \
    --backup-dir="$REMOTE/deleted/$(date +%Y%m%d)" \
    --delete-before \
    2>&1 | tee -a $LOGFILE

if [ ${PIPESTATUS[0]} -eq 0 ]; then
    echo "[$(date)] 同步成功" | tee -a $LOGFILE
else
    echo "[$(date)] 同步失败，发送告警邮件" | tee -a $LOGFILE
    # 发送告警邮件逻辑
fi
```

## 定时任务配置

### Cron定时备份

```bash
# 编辑定时任务
crontab -e

# 每天凌晨2点执行本地备份
0 2 * * * /root/backup.sh >> /var/log/backup_cron.log 2>&1

# 每6小时执行一次云端同步
# */6 * * * * /root/cloud_backup.sh >> /var/log/rclone_cron.log 2>&1

# 每周日凌晨执行完整同步
# 0 3 * * 0 rclone sync /data aliyun:weekly-backup --delete-before
```

### 系统D备份（推荐）

```bash
# 创建备份服务 /etc/systemd/system/backup.service

[Unit]
Description=NAS Backup Service
After=network-online.target

[Service]
Type=oneshot
User=root
ExecStart=/root/backup.sh
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target

# 创建定时器 /etc/systemd/system/backup.timer

[Unit]
Description=Run Backup Daily

[Timer]
OnCalendar=*-*-* 02:00:00
Persistent=true

[Install]
WantedBy=timers.target

# 启用定时器
systemctl enable backup.timer
systemctl start backup.timer
```
