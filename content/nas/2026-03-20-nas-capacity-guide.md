# 家庭NAS买几TB合适？2026年容量选购指南

## 前言

"买多大的硬盘？"这是每个NAS新手都会问的问题。买小了不够用，买大了又浪费钱。今天帮你彻底理清思路。

## 一、先算账：你的数据有多大？

### 1.1 各类型数据大小

| 数据类型 | 单文件大小 | 1000个文件 |
|----------|------------|-------------|
| 手机照片 | 3-8MB | 3-8GB |
| 高清电影 | 1.5-4GB | 1.5-4TB |
| 音乐专辑 | 30-100MB | 30-100GB |
| 文档资料 | 1-10MB | 1-10GB |
| 游戏资源 | 10-50GB | 10-50GB |

### 1.2 家庭容量估算公式

```
推荐容量 = (照片存储 + 电影存储 + 文档存储 + 其他) × 1.5（冗余）
```

## 二、不同场景容量推荐

### 2.1 轻度用户（手机备份+文档）

**使用场景**：
- 手机照片自动备份
- 文档资料同步
- 偶尔下载电影

**推荐容量**：**4TB**

```
4TB ≈ 可存100万张照片 或 1000部高清电影
```

**RAID建议**：RAID1（双盘备份）

**配置示例**：
```yaml
# 4TB RAID1 配置
drives:
  - /dev/sda  # 4TB
  - /dev/sdb  # 4TB
  
pool:
  layout: mirror
  size: 4TB  # 可用容量
```

### 2.2 中度用户（家庭影音中心）

**使用场景**：
- 电影、音乐收藏
- 家人照片集中管理
- PT下载挂机
- 轻度视频剪辑

**推荐容量**：**8TB**

```
8TB ≈ 可存2000部高清电影 + 200万张照片
```

**RAID建议**：RAID5（3-4盘）

**配置示例**：
```yaml
# 8TB RAID5 配置（4盘位）
drives:
  - /dev/sda  # 8TB
  - /dev/sdb  # 8TB
  - /dev/sdc  # 8TB
  - /dev/sdd  # 8TB
  
pool:
  layout: raid5
  size: 24TB  # 可用容量
```

### 2.3 重度用户（影视工作室）

**使用场景**：
- 4K视频素材存储
- 多相机RAW照片库
- 团队协作共享
- 7×24小时运行

**推荐容量**：**16TB+**

```
16TB+ ≈ 可存500小时4K视频 或 百万张RAW照片
```

**RAID建议**：RAID6（4-6盘）

### 2.4 极客用户（All in One）

**使用场景**：
- NAS + 软路由 + Home Assistant
- Docker/虚拟机运行
- PT保种
- 敏感数据加密

**推荐容量**：**16-32TB**

## 三、容量选购决策表

| 家庭人数 | 使用强度 | 推荐容量 | 硬盘数量 | RAID |
|----------|----------|----------|----------|------|
| 1人 | 轻度 | 4TB | 2 | RAID1 |
| 2人 | 轻度 | 4-8TB | 2 | RAID1 |
| 3-4人 | 中度 | 8-16TB | 4 | RAID5 |
| 5人+ | 中度 | 16TB+ | 6 | RAID6 |
| 工作室 | 重度 | 32TB+ | 8 | RAID6 |

## 四、为什么建议"买大不买小"？

### 4.1 硬盘价格规律

```
每TB单价 = 硬盘总价 ÷ 容量

2TB单盘 ≈ 400元 → 200元/TB
4TB单盘 ≈ 700元 → 175元/TB
8TB单盘 ≈ 1300元 → 162元/TB
16TB单盘 ≈ 2500元 → 156元/TB
```

**结论**：大容量硬盘每TB单价更低

### 4.2 数据增长速度

- 手机照片：每年增加 50-100GB
- 电影收藏：每年增加 200-500GB
- 工作资料：每年增加 50-200GB

**建议**：按3年后的需求选容量
