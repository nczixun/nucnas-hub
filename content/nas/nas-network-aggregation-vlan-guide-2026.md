# NAS 网络配置避坑指南：双网口聚合 + VLAN 隔离实战


很多用户买了双网口<a href="/nas/" target="_blank">NAS</a>，却只插了一根网线。2.5G 口当 1G 用，白白浪费性能。

更关键的是，智能家居设备越来越多，IoT 设备不隔离，安全隐患大。今天手把手教你配置链路聚合和 VLAN 隔离。

## 测试环境

| 设备 | 型号 | 网口 |
|------|------|------|
| NAS | 群晖 DS923+ | 双 2.5G 网口 |
| 交换机 | 威联通 QSW-308-1C | 8×2.5G + 1×10G SFP+ |
| 路由器 | 爱快 IK-Q3000 | 双 2.5G 网口 |
| 客户端 | 台式机 | 2.5G 有线 |

## 一、链路聚合配置（LACP）

### 1.1 交换机端配置

登录交换机管理界面：

```
1. 进入 端口管理 → 聚合组
2. 创建聚合组 LAG1
3. 添加端口 1、2 到 LAG1
4. 模式选择 LACP（802.3ad）
5. 应用保存
```

### 1.2 群晖端配置

DSM 7.2 操作路径：

```
控制面板 → 网络 → 网络界面 → 新增 → 创建 Bond
```

| 配置项 | 推荐设置 |
|--------|----------|
| Bond 模式 | IEEE 802.3ad 动态链路聚合 |
| 成员接口 | LAN 1 + LAN 2 |
| IP 设置 | 静态 IP（如 192.168.1.100） |
| 子网掩码 | 255.255.255.0 |
| 网关 | 192.168.1.1 |

**注意**：路由器必须支持 LACP，否则无法生效。爱快、OpenWrt、UBNT 都支持。

### 1.3 性能测试

聚合前后对比（SMB 传输）：

| 场景 | 单网口 | 双网口聚合 | 提升 |
|------|--------|------------|------|
| 单客户端 | 280 MB/s | 285 MB/s | 2% |
| 多客户端并发 | 280 MB/s | 520 MB/s | 86% |

**结论**：链路聚合不会提升单客户端速度（受限于单网口上限），但多客户端并发时总带宽翻倍。

## 二、VLAN 隔离配置

### 2.1 为什么要隔离 IoT 设备？

- 智能摄像头、插座安全性差，容易被入侵
- 一旦沦陷，会扫描内网其他设备
- 隔离后，IoT 设备无法访问 NAS、电脑等核心设备

### 2.2 交换机 VLAN 配置

```
1. 进入 VLAN 管理 → 创建 VLAN
2. VLAN 10：IoT 设备（192.168.10.0/24）
3. VLAN 20：核心设备（192.168.20.0/24）
4. VLAN 30：访客网络（192.168.30.0/24）
```

端口分配：

| 端口 | VLAN 模式 | PVID | 允许 VLAN |
|------|-----------|------|-----------|
| 1（NAS） | Trunk | 1 | 10,20,30 |
| 2（AP） | Trunk | 1 | 10,20,30 |
| 3-8（设备） | Access | 10/20/30 | 对应 VLAN |

### 2.3 群晖多 VLAN 配置

DSM 支持在一个物理网口上创建多个 VLAN 接口：

```
控制面板 → 网络 → 网络界面 → 新增 → 创建 VLAN
```

| VLAN ID | 用途 | IP 段 |
|---------|------|-------|
| 10 | IoT 管理 | 192.168.10.100 |
| 20 | 核心服务 | 192.168.20.100 |
| 30 | 访客访问 | 192.168.30.100 |

### 2.4 防火墙规则

在路由器或防火墙上设置：

```
允许：
- VLAN 20 → VLAN 10/20/30（核心设备可访问所有）
- VLAN 30 → 仅互联网（访客只能上网）

禁止：
- VLAN 10 → VLAN 20（IoT 不能访问核心设备）
- VLAN 30 → VLAN 10/20（访客不能访问内网）
```

## 三、常见问题排查

### 问题 1：聚合后速度没提升

**原因**：
- 路由器不支持 LACP
- 交换机聚合模式选错（选成静态聚合）
- 客户端也是单网口

**解决**：
```bash
# 群晖 SSH 登录，检查聚合状态
cat /proc/net/bonding/bond0
```

查看 `Slave Interface` 是否都显示 `up`，`MII Status` 是否为 `up`。

### 问题 2：VLAN 配置后无法上网

**原因**：
- 路由器没配置对应 VLAN 的 DHCP
- 网关设置错误

**解决**：
- 在路由器上为每个 VLAN 创建 DHCP 服务器
- 确保网关指向路由器对应 VLAN 的 IP

### 问题 3：NAS 跨 VLAN 访问慢

**原因**：
- 路由器性能瓶颈（软路由建议 4 核以上）
- 防火墙规则过于复杂

**解决**：
- 简化防火墙规则，只保留必要的禁止规则
- 升级路由器硬件

## 配置总结

|--------|--------|------|----------|
| 链路聚合 | 中 | ⭐⭐ | ★★★★ |
