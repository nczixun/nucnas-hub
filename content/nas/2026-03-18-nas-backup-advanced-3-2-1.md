# NAS备份策略进阶：3-2-1原则实战指南

## 什么是3-2-1备份原则？

3-2-1原则是数据备份领域的黄金法则：
- **3**：至少保留3份数据副本
- **2**：使用2种不同的存储介质
- **1**：其中1份存储在异地

对于NAS用户来说，这意味着你的重要数据不应该只依赖RAID阵列。

## 家庭NAS备份实战方案

### 第一层：本地RAID保护

RAID提供的是数据冗余而非备份。以下是推荐配置：

| RAID级别 | 冗余能力 | 可用容量 | 适用场景 |
|----------|----------|----------|----------|
| RAID 1 | 1块盘 | 50% | 重要文档 |
| RAID 5 | 1块盘 | n-1 | 家庭媒体 |
| RAID 6 | 2块盘 | n-2 | 企业级 |
| 静态存储 | 无 | 100% | 大容量仓库 |

### 第二层：本地备份副本

使用rsync或Duplicati在NAS内部创建第二份副本：

```bash
# rsync备份脚本示例
rsync -avz --delete /volume1/data /volume2/backup/
```

推荐方案：
- **备份软件**：Duplicati、UrBackup
- **备份频率**：每日增量，每周全量
- **保留策略**：保留30天历史版本

### 第三层：异地备份

| 方案 | 速度 | 成本 | 安全性 |
|------|------|------|--------|
| 公有云对象存储 | 慢 | 低 | 高 |
| 另一个NAS | 快 | 高 | 极高 |
| 冷备份（硬盘） | 快 | 中 | 中 |

推荐组合：
- **方案A**：NAS + 百度网盘/阿里云OSS
- **方案B**：双NAS异地同步
- **方案C**：NAS + 离线硬盘（定期备份）

## 进阶备份策略

### 版本控制

开启快照功能是保护数据的最后防线：

| 系统 | 快照工具 | 频率建议 |
|------|----------|----------|
| TrueNAS | ZFS Snapshots | 每小时 |
| Unraid | Unraid Snapshots | 每日 |
| FNOS | 快照功能 | 每小时 |
| PVE | LVM Snapshots | 每周 |

### 增量备份技巧

使用Restic或Borg Backup实现增量备份：

```bash
# Restic备份示例
restic -r /backup/repo backup /data
restic -r /backup/repo forget --keep-daily 7 --keep-weekly 4
```

## 常见误区

1. **RAID等于备份**：RAID只保护硬件故障，不保护误删除和勒索病毒
2. **只备份一次**：建议设置自动备份任务
3. **忽视加密**：异地备份务必加密，防止数据泄露

## 总结

3-2-1原则是数据安全的底线。建议家庭用户至少实现：
- 本地RAID 5/6
- 每日增量备份到第二块硬盘
- 每周同步到公有云
