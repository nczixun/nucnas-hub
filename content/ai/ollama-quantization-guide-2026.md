# Ollama 模型量化指南：显存节省 70% 的正确姿势

模型太大，显存不够用？量化是解决问题的关键。这篇讲清楚 Ollama 量化的原理和正确姿势。

## 什么是模型量化？

量化就是用更少的比特表示模型参数。简单说：用"低配"精度换"低配"显存。

| 精度 | 缩写 | 显存占用 |
| :--- | :--- | :--- |
| FP32 | 32位浮点 | 100% |
| FP16 | 16位浮点 | 50% |
| INT8 | 8位整数 | 25% |
| INT4 | 4位整数 | 12.5% |

## Ollama 支持的量化格式

### 官方量化后缀

```bash
ollama pull llama3.2:3b        # 默认 FP16
ollama pull llama3.2:3b-q4_0    # Q4_0
ollama pull llama3.2:3b-q4_1   # Q4_1
ollama pull llama3.2:3b-q5_0   # Q5_0
ollama pull llama3.2:3b-q5_1   # Q5_1
ollama pull llama3.2:3b-q8_0   # Q8_0
```

### 推荐量化等级

| 后缀 | 压缩率 | 质量 | 推荐场景 |
| :--- | :--- | :--- | :--- |
| q4_0 | ~60% | 较好 | 显存紧张首选 |
| q4_1 | ~55% | 更好 | 平衡选择 |
| q5_0 | ~45% | 很好 | 日常对话 |
| q5_1 | ~40% | 接近无损 | 重要任务 |
| q8_0 | ~25% | 无损 | 精度优先 |

## 如何选择量化版本

### 按硬件选择

| 显存/内存 | 推荐模型 |
| :--- | :--- |
| 4GB | 1B-3B + Q4 |
| 8GB | 3B-7B + Q4/Q5 |
| 16GB | 7B-14B + Q5 |
| 32GB+ | 14B+ + Q8/FP16 |

### 按使用场景选择

- **日常聊天**：Q4_0-Q5_0 足够
- **代码生成**：Q5_1 以上，防止幻觉
- **长文写作**：Q5_1，保持上下文
- **专业分析**：Q8_0 或 FP16

# 示例命令

## 常见问题

**Q：量化会丢失多少能力？**
A：Q5_1 以上基本无感知，Q4_0 损失约 10%。

**Q：N100 能跑多大模型？**
A：建议 3B Q4_0 或 1.5B Q4。

**Q：量化后能微调吗？**
A：可以，但建议用原始精度做微调。

**Q：不同量化能混合使用吗？**
A：可以，但没意义，保持统一。

## 最佳实践

1. **先试 Q5_1**：大多数场景足够
2. **不够再降**：Q4_0 质量也还行
3. **不要迷信 FP16**：除非显存真的够
4. **关注中文优化**：Qwen 系列中文效果更好

模型量化是让大模型在消费级硬件上跑起来的关键技术。Q5_1 是性价比最高的选择，Q4_0 则是显存紧张时的救星。

掌握量化技术，你的迷你主机和 NAS 也能跑动本地大模型。
