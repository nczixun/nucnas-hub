# Ollama 模型量化指南：如何让大模型在迷你主机上流畅运行

很多玩家想在 NUC 或迷你主机上跑本地大模型，但 16GB 内存根本跑不动 70B 参数的模型。今天聊聊模型量化，让你的小机器也能跑 Qwen、DeepSeek。

## 什么是模型量化

简单说，量化就是"压缩模型"。原始模型用 FP16（16位浮点数），量化后可以用 Q4_K_M（4位）甚至 Q2_K（2位）。

内存占用对比（以 Qwen2.5 14B 为例）：
- FP16：28GB 显存
- Q8_0：14GB 显存
- Q4_K_M：7.8GB 显存
- Q2_K：4GB 显存

理论上 16GB 内存的迷你主机也能跑 14B 的 Q4_K_M 量化版本。

## 常用量化版本一览

| 量化级别 | 内存占用 | 质量损失 | 推荐场景 |
|---------|---------|---------|---------|
| Q8_0 | ~14GB | 无明显损失 | 显存充足的玩家 |
| Q5_K_M | ~9GB | 轻微 | 追求平衡 |
| Q4_K_M | ~7.5GB | 可接受 | 16GB 内存首选 |
| Q3_K_M | ~6GB | 明显 | 极限压缩 |
| Q2_K | ~4GB | 较大 | 仅体验用 |

## 实战：部署量化模型

### 1. 查看可用模型
```bash
ollama list
```

### 2. 拉取指定量化版本
```bash
# 拉取 Qwen2.5 14B Q4_K_M 版本
ollama pull qwen2.5:14b-q4_K_M

# 拉取 DeepSeek R1 Q4_K_M 版本
ollama pull deepseek-r1:14b-q4_K_M
```

### 3. 运行时指定上下文长度
```bash
ollama run qwen2.5:14b-q4_K_M --ctx 4096
```

## 实测数据对比

测试机型：零刻 SER8（AMD 8845HS，32GB 内存）

| 模型 | 量化级别 | 首token响应 | 吞吐量 | 内存占用 |
|------|---------|------------|--------|---------|
| Qwen2.5 14B | Q8_0 | 2.1s | 18 tok/s | 16GB |
| Qwen2.5 14B | Q4_K_M | 1.4s | 25 tok/s | 8GB |
| Qwen2.5 14B | Q2_K | 0.8s | 35 tok/s | 4.5GB |
| DeepSeek R1 8B | Q4_K_M | 1.2s | 28 tok/s | 5GB |

## 量化模型的适用场景

**Q4_K_M 适合：**
- 日常对话、问答
- 代码辅助编程
- 文案撰写润色

**Q2_K 适合：**
- 体验尝鲜
- 内存极端受限的场景
- 不需要精确答案的场景

## 避坑指南

1. **不是越量化越好**：Q2_K 虽说 4GB 就能跑，但回答质量下降明显，逻辑错误增加
2. **部分模型不支持所有量化**：可以用 `ollama show 模型名` 查看可用版本
3. **Mac 用户注意**：Metal 加速对量化支持有限，Intel Mac 建议用 CPU 量化版本
4. **CPU 推理也能用**：纯 CPU 模式下，Q4_K_M 在 32GB 内存的 NUC 上也能跑，但速度较慢

## 一句话总结

16GB 内存选 Q4_K_M，32GB 内存选 Q8_0，极限压缩选 Q2_K 体验一下。量化不是万能的，但在硬件有限的情况下，这是让本地大模型可用的最佳方案。
